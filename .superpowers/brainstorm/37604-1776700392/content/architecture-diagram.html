SSE Notification Flow
1. Upload → FastAPI saves doc, returns {id}
2. Frontend opens GET /api/documents/{id}/status
3. FastAPI polls DB every 2s, streams data: {status}
4. Worker finishes → updates status to done
5. SSE sends final event → frontend shows toast
docker-compose Services
frontend — nginx serving built React
backend — FastAPI (uvicorn)
worker — Celery (same image, different cmd)
redis — Redis 7
db — pgvector/pgvector image
